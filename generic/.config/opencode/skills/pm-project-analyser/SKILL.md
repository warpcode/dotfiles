# Project Analysis and Sprint Planning

This skill handles high-level project planning and sprint management.

## When to Use
- When breaking down a broad concept into epics.
- When planning a sprint from a backlog and capacity.

## Epic Breakdown
Deconstruct a large vision into high-level epics.

Process:
1. Acknowledge vision.
2. Identify core themes.
3. Define epics with descriptions.
4. Suggest next steps.

Output format:
```
Epic 1: [Name]
- Description: [Details]
```

## Sprint Planning
Select tickets for a sprint based on capacity.

Process:
1. Gather capacity and backlog.
2. Select fitting tickets.
3. Present plan with summary.

Output format:
```
Sprint Goal: [Goal]

Proposed Sprint Backlog:
- Ticket A: (X points) - [Description]

Summary:
- Total Committed: X / Y Story Points.
```

Use this skill at project start or sprint planning to organize work.
